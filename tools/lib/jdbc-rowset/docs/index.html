JDBCTM RowSet Implementations provides an API and Synchronization SPI that standardardize the RowSet interface implementations originially introduced in JDBC 3.0 API Specification.

The JDBC RowSet specification and reference implementation provides a set of RowSet implementation definitions that allow tabular datasources such as a SQL Database to be exposed as convenient JavaBeansTM components.

JSR-114, defines RowSet interfaces that provide the following functionality;

  • A continuously connected RowSet for immediately SQL database manipulation
  • The ability for a RowSet to sever a connection from the data source and to easily synchronize it's contents with a persistent datastore using pluggable Synchronization Providers.
  • Manage large sets of data efficiently by paging specific tabular rows as needed.
  • Articulate the contents of a RowSet in a standard XML format according to the WebRowSet schema.
  • Manipulate the contents of a RowSet using standardized filtering and SQL JOIN RowSet interface definitions and mechanisms.

JDBC RowSets give developers an efficent and standard mechanism to handle tabular data structues in both disconnected and connected environment. By providing sufficient tools to mapulate transactional SQL datasources with the added convenience of a pluggable SPI to better manage the complexities of disconnected syncrhonization.
